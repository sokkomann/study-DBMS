TCL(Transaction Control Language): 트랜젝션 제어어
	트랜젝션이란, 하나의 서비스를 위한 DML의 묶음 혹은 작업 단위를 말함.
	
	예를 들어, "상품 구매 시 포인트 적립" 서비스는 "상품 구매 서비스"이다.
	이 서비스에 필요한 최소 DML 개수는 2개(INSERT, UPDATE)이다.
	그렇다면 상품 구매 서비스의 트랜잭션은 INSERT와 UPDATE이다.
	이 때, INSERT는 잘 되었어도 UPDATE에서 문제가 발생하면, 
	다시 INSERT를 복구해줘야 하기 때문에 TCL 명령어를 배워야 한다.

DML의 delete를 사용하면 TCL의 롤백으로 복구가 되지만
DDL의 truncate을 사용하면 복구가 안됨. 왜냐면 TCL이란게 DML의 묶음이니까

	commit   
		현재 트랜잭션의 모든 작업을 확정하는 명령어
		한번 커밋하면 되돌릴수없어. 그렇다고 쫄지말고 단순히
		오류안나면 당연히 commit 실행

	rollback
		이전 커밋한 지점으로 이동하는 명령어
		오류나면 당연히 rollback 실행

	※ DML만 가능하고, DDL은 해당사항 없음.
	따라서 DDL은 신중히 작성해야 함.(create drop alter truncate)

View	DML로 정규화된 테이블들을 join해서 만든 가상의 테이블을 View로 저장해서 보자

	기존의 테이블은 그대로 놔둔채 필요한 컬럼들 및 새로운 컬럼들을 만든 가상 테이블.
	실제 데이터가 저장되는것은 아니지만 View를 통해서 충분히 데이터를 관리할 수 있다.
	※ 조회만 하자!(View로 만들어진 가상의 테이블을 추가,수정,삭제 말고 조회만 하자는거지)
	그니까 편하게 조회하려고 View를 쓰는거임

	- 독립성: 다른 곳에서 접근하지 못하도록 하는 성질
	- 편리성: 긴 쿼리문을 짧게 만드는 성질
	- 보안성: 기존의 쿼리문이 남들에게 보이지 않는다. 

View 문법
	- 새롭게 생성
		create view [뷰 이름] as (select 쿼리문);

	- 새롭게 생성 혹은 기존 View 수정
		create or replace [뷰 이름] as (select 쿼리문);
